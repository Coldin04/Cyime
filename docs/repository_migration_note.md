# 仓库迁移与提交历史说明

## 背景

在整理部署流程的过程中，本仓库曾误将部分构建产物提交进版本历史。

后续在清理这些无关文件、整理分支并迁移到 GitHub 时，部分提交经历了重新提交与重新签名，因此当前 GitHub 仓库中的部分 commit 哈希、提交时间和签名展示，与早期仓库记录可能不完全一致。

## 需要说明的事实

- 当前 GitHub 仓库内容为清理后的有效版本。
- 部分 commit 因历史整理而重新生成了新的哈希。
- GitHub 上显示的部分签名状态或签名时间，可能与早期仓库中的表现不同。
- 这次变化来自历史整理与迁移过程，不代表项目作者或维护身份发生变化。
- 使用的签名密钥仍然是同一把密钥，并未更换。

## 为什么会出现这种情况

这次历史变化主要来自两个动作：

1. 清理误提交的构建产物
2. 在迁移到 GitHub 的过程中重新整理提交

因此，旧仓库中的 commit 记录与当前 GitHub 仓库中的 commit 记录，并不一定能逐条用哈希完全对应。

## 当前以哪个仓库为准

当前 GitHub 仓库应视为清理后的主代码仓库版本。

如果需要核对部署效果、查看公开展示版本或结合原有协作记录进行验证，可参考下方 Codeberg 公共部署仓库：

- Codeberg 部署仓库：`https://codeberg.org/Coldin04/CyimeWrite_archive`

## 对验证者的建议

如果需要验证项目当前状态，建议优先关注：

- 当前 GitHub 仓库中的代码内容
- 对应部署仓库中的运行版本
- README 与项目文档中的说明

不建议仅根据旧 commit 哈希是否完全一致来判断当前仓库内容的有效性。
